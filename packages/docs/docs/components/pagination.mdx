# Pagination

## Introduction

Using pagination to navigate pages of component listings. It helps users easily move between a large number of pages.

<img
  src={StbPaginationStructure1}
  srcSet={`${StbPaginationStructure2} 2x`}
  alt="Pagination structure"
/>
<ol className="u-textGray u-marginTopTiny">
  <li>
    Previous/Next
    <i className="u-text200"> - Navigate backward and forwards from the currently selected page.</i>
  </li>
  <li>
    Page navigation
    <i className="u-text200"> - Navigate to a specific page by selecting the page number.</i>
  </li>
  <li>
    Truncation
    <i className="u-text200"> - When a threshold of pages is reached, the list is truncated using ellipses.</i>
  </li>
</ol>

## Examples

The Previous button, Next button, First-page item, and Last-page item are always visible by default. The UI could be different depends on the number of pages.

### Sizes

Pagination component shares the same sizes as the Buttons component. However, buttons in pagination must be the same size.

<CodeView exampleClassName="example-paginations" codeText={sizesPagination} />

### Structure

#### Less than or equal to 7 items

If the first page is active, then the Previous button will be disabled. If the last page is active, then the Next button will be disabled.

<CodeView exampleClassName="example-paginations" codeText={lessThan7Pagination} />

#### More than 7 items

Except for the Previous and Next buttons, the total number of visible items is fixed to 7. While the first and last items are always visible, 5 extant items can move around.

<CodeView exampleClassName="example-paginations" codeText={moreThan7Pagination} />
