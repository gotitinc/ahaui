# Table

## Introduction

Due to the widespread use of tables across third-party widgets like calendars and date pickers, weâ€™ve designed our tables to be opt-in. Just add the base class  .Table to any `<table>`, then extend with custom styles or our various included modifier classes.

Responsive tables
Tables can be responsive by adding these CSS classes into their parent elements:
`.u-block .u-widthFull .u-overflowHorizontalAuto`

## Examples
### Default

Default style with white background and no border.

<CodeView codeText={defaultTable}/>

### Sticky Header

Fixed the header on top of the table during the vertical scrolling.

<CodeView codeText={stickyHeaderTable}/>


### Striped
